BZOJ 1218 [HNOI2003]激光炸弹

Posted on 2018-10-01 | Edited on 2018-10-06 | In Historic

Description
一种新型的激光炸弹，可以摧毁一个边长为R的正方形内的所有的目标。现在地图上有n(N<=10000)个目标，用整数Xi,Yi(其值在[0,5000])表示目标在地图上的位置，每个目标都有一个价值。激光炸弹的投放是通过卫星定位的，但其有一个缺点，就是其爆破范围，即那个边长为R的正方形的边必须和x，y轴平行。若目标位于爆破正方形的边上，该目标将不会被摧毁。

Input
输入文件的第一行为正整数n和正整数R，接下来的n行每行有3个正整数，分别表示xi,yi,vi

Output
输出文件仅有一个正整数，表示一颗炸弹最多能炸掉地图上总价值为多少的目标（结果不会超过32767）。

Sample Input
2 1

0 0 1

1 1 1
Sample Output
1

其实这道题可以用二维前缀和。sum[i][j]表示(0,0)到(i,j)的和。

计算完前缀和之后枚举每个点红色正方形全值=前缀和（绿色正方形）-两个长方形+蓝色正方形（蓝色正方形被减了两次）。最后取最大值即可。

在经过代码实现之后。。。我发现此题真的是个毒瘤题（或者说数据水），理由如下：

  1. 题面中说“若目标位于爆破正方形的边上，该目标将不会被摧毁。”。所以对于一个正方形威力圈来说，除了把中间的目标摧毁，还可以将一组邻边上的目标摧毁。（将威力圈向两个方向分别平移无限小）。对于邻边最优解的判断比较恶心，而且网上题解大部分没管也过了。。。。
  2. 不一定必须将威力圈全部放在矩阵内。很多人都是判断当前枚举边长是不是小于R的。然而数据狠一点可以卡住。所以需要特判。。。
  3. 内存只给一百多M。。。两个图都存不下，非要把元数据和前缀和存一起。。。麻烦死了。

综上所述，果断放弃。。。
